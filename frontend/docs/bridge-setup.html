Installer le bridge MT5

Le bridge est une petite application Windows qui tourne sur ton PC et relaie les ordres entre Scalping Radar et ton terminal MT5. Compter 10-15 minutes pour tout configurer.

Sommaire
  1. Prérequis
  2. Installer Tailscale
  3. Installer MetaTrader 5
  4. Télécharger et lancer le bridge

00Prérequis

  • Un PC Windows 10 ou 11 allumé pendant les heures où tu veux trader (les marchés forex tournent dimanche 22h UTC → vendredi 22h UTC)
  • Un compte broker supportant MT5 (par ex. Pepperstone, IC Markets, MetaQuotes, etc.)
  • Un compte Tailscale gratuit (pour exposer le bridge au cloud sans ouvrir de port public)
  • Un compte Scalping Radar actif
Sécurité

Aucun fond n'est chez nous. Le bridge tourne sur ton PC, connecté à ton broker via tes identifiants MT5. Scalping Radar envoie uniquement des instructions de trade (pair, direction, SL, TP) au bridge, qui les exécute localement. Tes clés broker ne transitent jamais par nos serveurs.

01Installer Tailscale

Tailscale crée un réseau privé virtuel (VPN mesh) entre ton PC et nos serveurs. C'est gratuit jusqu'à 3 appareils et sans configuration réseau (pas de port forwarding).

  1. Crée un compte sur tailscale.com (connexion via Google/GitHub/Microsoft)
  2. Télécharge et installe le client Windows depuis tailscale.com/download/windows
  3. Ouvre Tailscale et connecte-toi. Note l'adresse 100.x.y.z qui apparaît dans l'interface — c'est l'IP de ton PC dans le réseau privé

02Installer MetaTrader 5

Télécharge MT5 depuis le site de ton broker (pas depuis metaquotes.net, certains brokers livrent une build custom). Ensuite :

  1. Ouvre MT5 et connecte-toi avec tes identifiants broker (login + server + password investor ou master)
  2. Active Outils → Options → Expert Advisors → "Autoriser le trading algorithmique"
  3. Laisse MT5 ouvert tant que tu veux que le radar fonctionne
Démo d'abord

On recommande fortement de démarrer sur un compte démo (chaque broker en propose un). Tu valides le comportement du bridge sur plusieurs jours, puis tu passes en compte réel une fois confiant.

03Télécharger et lancer le bridge

Le bridge est un exécutable Windows qui parle à MT5 via le protocole MetaTrader. Il expose un serveur HTTP local protégé par API key.

  1. Télécharge la dernière version du bridge : scalping-bridge-win.zip
  2. Dézippe et lance bridge.exe
  3. Au premier lancement, le bridge génère automatiquement une API key aléatoire et l'affiche dans le terminal. Copie-la — tu en auras besoin à l'étape suivante
  4. Le bridge écoute par défaut sur http://0.0.0.0:8787

Tu peux vérifier qu'il tourne depuis n'importe quel onglet :

curl http://localhost:8787/health -H "X-API-Key: TA_CLE"
# → {"status": "ok", "mt5_connected": true, ...}

04Connecter Scalping Radar

Dernière étape : donner à Scalping Radar l'URL Tailscale de ton bridge + l'API key.

  1. Construis l'URL : http://<ton-ip-tailscale>:8787 (ex. http://100.64.0.12:8787)
  2. Va sur l'onboarding de Scalping Radar
  3. Colle l'URL + l'API key, clique "Tester la connexion"
  4. Si tu vois Bridge joignable ✓, tu es bon. Sinon voir dépannage
Ouvrir l'onboarding →

05Dépannage

Le test de connexion échoue (bridge injoignable)

  • Vérifie que Tailscale est bien connecté (icône dans la barre système Windows)
  • Vérifie que bridge.exe tourne toujours
  • Vérifie l'URL : il faut http:// (pas https), l'IP Tailscale (100.x.y.z, pas 192.168.x.x), et le port :8787
  • Depuis ton navigateur, essaie d'ouvrir http://<ton-ip-tailscale>:8787/health — tu devrais avoir un JSON

Le bridge tourne mais MT5 n'est pas "connecté"

  • Ouvre MT5, connecte-toi à ton compte
  • Vérifie Outils → Options → Expert Advisors : "Autoriser le trading algorithmique" doit être coché
  • Redémarre le bridge après avoir configuré MT5

Ordres rejetés pour "invalid stops"

  • Certains brokers exigent un stop_level minimum (distance mini entre entry et SL/TP). Scalping Radar respecte ce paramètre, mais en cas de marché ultra-volatil le rejet peut arriver
  • Consulte la carte Rejections sur le cockpit (Pro+) pour voir les motifs

Le bridge consomme trop de CPU / RAM

  • C'est normalement très léger (< 50 Mo RAM, < 1% CPU au repos). Si tu vois des pics, ouvre une issue sur support@scalping-radar.com avec une capture

Autre problème ?

Écris-nous à support@scalping-radar.com avec les logs du bridge (console) et une description du comportement. Réponse sous 24h en semaine.
